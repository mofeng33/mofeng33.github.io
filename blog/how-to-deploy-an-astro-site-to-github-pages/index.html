如何将Astro站点部署到GitHub Pages上

可以使用 GitHub Actions 自动构建和部署，将 Astro 站点部署到 GitHub Pages。

请注意，您的仓库必须是公开的，才能在 GitHub 页面上发布您的网站。

  1. 设置astro.config.mjs
import { defineConfig } from 'astro/config'

export default defineConfig({
  site: 'https://<YOUR_USERNAME>.github.io',
  base: '/my-repo',
})

site应为 或https://<YOUR_USERNAME>.github.io 或 https://my-custom-domain.com base应该是以正斜杠开头的仓库名称，例如。这是为了让 Astro 了解您网站的根目录的位置，而不是默认路径。/my-repo/my-repo/

注意 在以下情况下，请勿设置参数：base

  • 您的存储库名为 。<YOUR_USERNAME>.github.io
  • 您使用的是自定义域。
  1. 在你的github项目仓库的根目录中创建一个名为.github/workflows的新文件夹，然后再新建一个deploy.yml文件，将以下内容粘贴YAML中。 .github/workflows/deploy.yml

name: Deploy to GitHub Pages

on:
  # Trigger the workflow every time you push to the `main` branch
  # Using a different branch name? Replace `main` with your branch’s name
  push:
    branches: [ main ]
  # Allows you to run this workflow manually from the Actions tab on GitHub.
  workflow_dispatch:

# Allow this job to clone the repo and create a page deployment
permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout your repository using git
        uses: actions/checkout@v3
      - name: Install, build, and upload your site
        uses: withastro/action@v0
        # with:
            # path: . # The root location of your Astro project inside the repository. (optional)
            # node-version: 16 # The specific version of Node that should be used to build your site. Defaults to 16. (optional)
            # package-manager: yarn # The Node package manager that should be used to install dependencies and build your site. Automatically detected based on your lockfile. (optional)

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v1

  1. 在 GitHub 上，转到仓库的“Settings”选项卡，然后找到设置的“Pages”部分。

  2. 选择“GitHub Actions”作为你网站的源。

  3. 将你本地的更新推送到 GitHub。

接下来可以通过https://<YOUR_USERNAME>.github.io 或 https://my-custom-domain.com 访问你的站点了

注意： 一定要将.github/workflows/deploy.yml文件放在你的GitHub仓库的项目根目录下，不然你的网站内容不会得到更新。
