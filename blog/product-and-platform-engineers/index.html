产品和平台工程师

前端和后端工程师之间的鸿沟越来越不明显：

  • 前端开发人员不再只是编写HTML，CSS和JavaScript。现在，通常可以看到专注于前端的开发人员从头开始构建整个 Web 应用程序。它们集成并连接到数据库、处理身份验证等。
  • 后端开发人员通常被迫在两条路径之间进行选择：编写更多的前端代码来支持他们的后端，或者更接近于管理、维护和构建前开发人员使用的软件基础架构。
  • 人工智能增强的开发人员已经将重点从简单的编写代码转移到创建出色的产品。初创公司越来越感受到这种转变，他们运行着一个更精益的工程团队，使用LLM和编码副驾驶来更快地行动。

考虑两个不同的垂直行业：产品和平台工程师。

什么是产品工程师？

产品工程师从所需的产品体验向后工作到支持它的一组技术。他们考虑前端、后端、设计以及介于两者之间的一切，以创造出色的用户体验。

他们不需要深入了解每个部分，这是对“全栈”的常见误解。相反，他们对可用的工具有广泛的了解，并在应用这些工具来构建产品方面拥有丰富的经验。

优秀产品工程师的一些素质：

迭 代：他们重视迭代速度和增量正确性。他们宁愿发货，从客户那里获得反馈，并在此过程中进行调整，而不是在孤岛中构建。 以客户为中心：他们不怕与客户互动并学习如何改进产品。在一家较大的公司，产品经理可能专注于此，但应与工程师密切合作，将这些知识应用于正确的解决方案。 务实：他们明白，技术选择都是达到目的的手段。如果没有人使用您的产品，选择最新的 JavaScript 库并不重要。如果他们没有帮助实现他们的产品目标，他们愿意删除工具。 好吧，那么另一半软件工程师会留下什么呢？

产品与平台工程

产品工程师专注于构建和增强解决最终用户问题的功能，而平台工程师则专注于支持产品的基础架构。

平台工程团队正在构建或购买工具来支持产品工程师，并使他们更高效、更高效、更快乐。他们通常会为产品团队做出基础结构选择。

通常，平台团队仅供大型组织访问。越来越多的小而有效的平台工程师团队可以做出令人难以置信的有影响力的决策，即使对于较小的团队也是如此。他们深思熟虑地评估和购买软件，以随着产品团队的规模而增长。

寻找人才

大多数初创公司都在寻找全栈工程师，但实际上需要产品工程师。 最好的产品工程师有两个共同的品质：

对打造高品质体验的热情 不断学习和探索新想法的动力 他们不是像素推动者;他们是为自己的工作感到自豪的创作者，并且不怕支持他们的产品。他们应该有很多工作的例子，因为其中很多都是面向用户的。

他们了解Web基础知识，例如浏览器，可访问性，动画，CSS，HTML和JavaScript。他们使用的工具——无论是 React、Svelte 还是其他工具——都是次要的。真正重要的是他们使用这些工具来构建最佳产品的能力。

💡从全栈→产品工程师的头衔转变有一些先验技术。 设计也发生了类似的转变。像UI / UX设计师和平面设计师这样的头衔越来越多地被产品设计师所取代。

伟大的产品工程师不会止步于视觉上的完整。他们仔细考虑用户旅程。如果互联网连接速度慢（或没有）会怎样？如果此 API 返回错误怎么办？此体验在触摸设备上如何工作？我们应该争取什么延迟数字？

对于这些开发人员来说，对其他人来说似乎是过多的润色是正常的。

结论

当我在推特上发布“产品工程师>全栈工程师”时，我对回应感到惊讶。

在Vercel，我们更新了我们的职位描述，将参考从Fullstack更改为产品工程师。我注意到其他人也在做同样的事情。

产品工程师代表了Web开发的转变。特别是随着人工智能的兴起，越来越需要不仅仅是“编写代码”的开发人员 - 了解大局，可以跨各种技术工作，并专注于为最终用户提供卓越产品的开发人员。
