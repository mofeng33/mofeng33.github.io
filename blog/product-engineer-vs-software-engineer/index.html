产品工程师与软件工程师的区别

软件公司正在发生变化。曾经产品经理和软件工程师占主导地位，现在出现了一个新的角色：产品工程师。

去年，产品工程师的在线搜索量增长了80%以上，现在比两年前增长了3倍。

产品工程师正在改变公司对工程的思考方式。为了帮助解释原因，我们将产品工程师的角色与传统的（仍然是非常必要的）全栈软件工程师的角色进行比较。

产品和软件工程师有何不同？

简单概括就是：

  • 产品工程师专注于构建出色的产品。
  • 软件工程师专注于构建出色的软件。
  • 但他们都编写和提交代码。

让我们进一步分解这些要点。

产品工程师

  • 有时被称为具有产品意识的工程师
  • 拥有产品，并对其成功和失败负责。 他们对用户感同身受，关心解决他们的问题。他们在构建时考虑到了用户，并创造了用户成功所需的东西。

相比之下，软件工程师拥有他们编写的代码。他们更关心解决技术问题、优化系统以及编写干净、安全的代码。他们在实施方面具有灵活性，但他们不做出产品决策。

产品工程师较少关注实现，而更专注于解决用户的问题。这样，他们是务实的。如果需要，他们愿意快速构建、迭代甚至从头开始构建。

软件工程师更理想化。他们寻找摆在他们面前的问题的最佳解决方案，专注于最佳实践，并在先前的工作基础上进行构建。安全性、风险和扩展在他们心目中更加重要。

增长工程师呢？ 增长工程师是一种软件工程师，专注于推动增长指标的项目，如注册、订阅和收入。他们与产品工程师有许多共同的特点，但更广泛地关注整个公司的增长，而不是特定的产品。

他们如何度过自己的时间？

产品工程师仍然花费大部分时间进行编码，但他们在编码之外承担着软件工程师没有的责任。

产品工程师还花时间与客户交谈，挖掘使用数据并研究竞争格局。他们测试其他产品，构建原型，并进行头脑风暴实验，以使他们的产品更好。

他们花时间与用户、他们的团队和公司的其他成员进行沟通。他们交流他们在研究中学到的东西，为什么他们的工作与该研究相关，以及如何确定下一步工作的优先级。他们拥有产品路线图，这伴随着沟通的责任。

软件工程师专注于编写代码、测试和维护代码。他们寻找优化代码、提高可伸缩性和解决错误的方法。他们专注于特定的技术领域，无论是数据库、数据管道、后端 API 还是客户端应用框架。这意味着研究和阅读文档、更新和代码。

为什么对产品工程师的需求在增长？

没有单一的原因，但通常是出于更快地交付产品改进的愿望。出于这个原因，产品工程师在初创公司和早期公司中很受欢迎，即使他们不说出来。许多工程师的行为就像产品工程师，即使这不是他们的职位。

在大公司，产品工程师被视为推动创新的一种方式，有时是通过创建可以自由构思和执行产品创意的小型团队。软件工程师和多学科团队仍然很重要，但产品工程师可以在需要的地方注入紧迫感。

埃隆·马斯克（Elon Musk）的Twitter是一个很好的例子，说明公司可以从产品工程师中受益，无论他是否意识到这一点。Elon 希望对产品进行重大更改，但在他看来，他继承的组织太大且笨拙，无法实现这一目标。

换句话说，他不需要软件工程师来维护或优化现有的代码库;他需要能够识别问题和机会，并快速独立执行解决方案的产品工程师。

产品工程师适合你吗？

优秀的产品工程师需要：

  • 以客户为中心。
  • 享受直接与用户交谈的乐趣。
  • 深切关注结果和影响。
  • 灵活实施。

如果这听起来像您，那么产品工程可能适合您。

但是，如果您想专注于解决困难的技术问题，并优化现有的技术和流程，那么更传统的软件工程角色是更好的选择。

请记住，产品工程更像是一种工作和思考方式，而不是一个明确的、僵化的角色。产品工程师应该足够灵活，以便在需要时深入进行技术实施。同样，软件工程师可以随时选择采用“产品工程师”的心态。

公司应该什么时候聘请产品工程师？

产品工程师非常适合初创公司和希望敏捷和快速行动的公司，例如 B2B SaaS 或消费者订阅服务。他们对产品有整体看法，并在需要时快速行动的能力。对于拥有大型、技术复杂产品（如数据工具或大型企业平台）的公司来说，它们的效果较差。

选择正确的事情取决于公司及其目标。如果你是一个专注于构建伟大产品的小团队，那么可以与用户交谈的多技能产品工程师是一个不错的选择。如果您的大型产品需要专业化来扩展和维护，软件工程师是更好的选择。具有这两种角色的公司已经成功，更多的公司将继续成功。
